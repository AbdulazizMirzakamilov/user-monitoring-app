Настольное приложение для комплексного мониторинга и анализа действий пользователей. Система ориентирована на специалистов по информационной безопасности и администраторов, предоставляя инструменты для выявления аномалий и контроля сетевой активности.

---

## ✨ Основной функционал

* **Аналитические отчеты** - генерация 5 видов специализированных выборок (аномалии, IP-адреса и непрерывность работы).
* **Аномальная активность** - выявление пользователей, превысивших лимит запросов за выбранный период.
* **Контроль IP-адресов** - детекция использования одной учетной записи на разных физических устройствах.
* **Непрерывная работа** - сложный анализ сессий для поиска работы без выходных.
* **Анализ активности по дням** - оперативный отчет за конкретную дату, показывающий суммарную нагрузку по каждому сотруднику.
* **Топ операций (Статистика)** - агрегированный отчет за период, отображающий распределение различных типов действий (поиск, ввод, корректировка).
* **Контейнеризация** - быстрое развертывание всей инфраструктуры базы данных через Docker.
* **Умный поиск** - мгновенная фильтрация данных по ФИО или логину.
* **Валидация** - встроенная система проверки корректности ввода данных.

## 🧠 Реализованные отчёты

### 1. Аномальная активность
Считает количество критических операций (например, обращений к БД). Позволяет администратору задать "порог" чувствительности, чтобы отсеять обычный трафик и сфокусироваться на подозрительных всплесках.

### 2. Анализ IP-адресов
Анализирует таблицу входов. Если за выбранный период у одного пользователя зафиксировано более одного уникального IP-адреса, система сигнализирует о возможной передаче учетных данных или несанкционированном доступе.

### 3. Активность по дням
Позволяет выбрать конкретную дату в календаре и получить список всех сотрудников, работавших в этот день, с указанием их общего количества действий. Идеально для быстрой ежедневной проверки.

### 4. Топ операций (Статистика)
Формирует сводную таблицу за период времени. Группирует данные по типам операций (`operation_types`), показывая, какие функции системы (поиск, ввод данных, удаление) использовались чаще всего.

### 5. Непрерывная работа
Использует **оконные функции SQL** (`ROW_NUMBER()` и `PARTITION BY`) для вычисления серий идущих подряд дней активности. Помогает контролировать трудовую дисциплину и выявлять аномально длинные рабочие серии.

## 🚀 Быстрый запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/AbdulazizMirzakamilov/user-monitoring-app
   ```
2. **Разверните базу данных:**
   В корне проекта (где находится файл `docker-compose.yml`) выполните команду:
   ```bash
   docker-compose up -d
   ```
3. **Настройте подключение:**
   Проверьте строку подключения в файле `App.config`. По умолчанию она настроена на работу с контейнером через `localhost:5432`.

4. **Запуск приложения:**
   Откройте файл решения (`.sln`) в Visual Studio и нажмите **F5** для сборки и запуска.
